# Stage 15 — Export Files

**Purpose:** Finalize editorial notes, build the ZIP bundle (if multi-section), and present the user with the run summary and next steps.

## Input

All artifacts produced by Stages 1–13, validated by Stage 14.

## Output

- `WORK_DIR/04-editorial-notes.md` — finalized from `_editorial-notes-draft.md`.
- `WORK_DIR/05-export.zip` — bundled output for multi-section sources, or skipped for single-section sources.
- Console summary printed to the user.

## Steps

### 1. Finalize editorial notes

Move `WORK_DIR/_editorial-notes-draft.md` to `WORK_DIR/04-editorial-notes.md` after enriching it with:

- **Header section:**
  ```
  # Editorial Notes — [Source Title]

  Generated by /podcast skill on [YYYY-MM-DD].
  
  This file records every change the skill made relative to the original
  source, every external reference used during enrichment, and every
  judgment call that needed human-reviewable provenance.
  ```

- **Sections in order:**
  1. **Source identification:** Title, author, content type, period, audience, detected tradition.
  2. **Cleaning summary:** What was normalized, deduplicated, repaired.
  3. **Foreign-term handling:** Scripts detected, terms substituted, new terms proposed.
  4. **Enrichments:** Every context bridge added (from Stage 10).
  5. **Analogies:** Every modern analogy added (from Stage 11).
  6. **Refinement decisions:** Notable tone-preservation choices.
  7. **Open questions:** Anything the skill flagged for user review but did not block on.

### 2. Build ZIP (multi-section only)

If `_segments/segments.yml` has more than one section, build `05-export.zip` containing:

```
[source-slug]/
  01-refined/
    section-01-<slug>.md
    section-02-<slug>.md
    ...
  02-instructions/
    podcast-instructions.md
  03-pronunciation.md
  04-editorial-notes.md
```

Excluded from ZIP:
- `00-source/` (original, not part of deliverable)
- `06-library-proposals.md` and `06b-borderline.md` (workflow artifacts, not deliverable)
- Any `_*` directories or files (working/intermediate state)
- `chapter-segmentation-proposal.md` (resolved before reaching this stage)
- `_quality-gates-report.md` (workflow artifact)

ZIP naming: `<source-slug>-podcast-export-<YYYY-MM-DD>.zip` inside the WORK_DIR.

### 3. Single-section sources

For single-section sources, no ZIP. The user can directly use the individual files in WORK_DIR. The summary names each file path.

### 4. Console summary

Print to the user:

```
================================================
/podcast complete — [Source Title]
================================================

Source-slug: [slug]
Content type: [type]
Detected tradition: [tradition (or "none")]
Sections: [N]
Pipeline: 15/15 stages complete
Quality gates: 5/5 passed

Outputs:
  ✓ Refined sections   → [WORK_DIR]/01-refined/
  ✓ NotebookLM         → [WORK_DIR]/02-instructions/podcast-instructions.md
  ✓ Pronunciation      → [WORK_DIR]/03-pronunciation.md
  ✓ Editorial notes    → [WORK_DIR]/04-editorial-notes.md
  ✓ Export ZIP         → [WORK_DIR]/05-export.zip   (multi-section)
  
Library proposals:
  [N] proposed additions to journal libraries staged.
  Review at: [WORK_DIR]/06-library-proposals.md
  
  To apply proposals, run:
      /podcast apply [source-slug]
  
  This will:
    – Show per-library diffs.
    – Let you accept all / accept some / reject.
    – Commit accepted changes with the source-slug in the message.
    – Re-run journal-side quality gates.

To paste into NotebookLM:
  1. Open NotebookLM.
  2. Upload each file from [WORK_DIR]/01-refined/ as a source.
  3. Use [WORK_DIR]/02-instructions/podcast-instructions.md as
     the customization prompt.
  4. Generate Audio Overview.

================================================
```

For single-section sources, adjust the file list accordingly.

### 5. Update editorial notes with apply pointer

Append to `04-editorial-notes.md` at the end:

```markdown
## Apply step

[N] library proposals are staged in `06-library-proposals.md`. They have
not been applied — `quotes-library.txt`, `clinic-library.txt`, and
`translations-glossary.md` are unchanged.

To apply: `/podcast apply [source-slug]`

To discard: delete `06-library-proposals.md` (the rest of the export
remains valid).
```

If no proposals were generated, append:

```markdown
## Apply step

No library proposals were generated from this source. Nothing to apply.
```

### 6. Clean up intermediate artifacts (optional)

The skill keeps all `_*` working files in WORK_DIR by default. The user can run `/podcast cleanup [source-slug]` separately to remove them and keep only the deliverable files. This is a separate command, not part of the export stage.

## What this stage does NOT do

- Does not write to journal libraries — that's the apply step.
- Does not delete WORK_DIR — that requires explicit user action.
- Does not regenerate any output — pipeline must complete cleanly through Stage 14.
