# Library Proposal Entry — Canonical Format for 06-library-proposals.md

The staging file `06-library-proposals.md` collects all proposed additions to journal libraries from a single podcast run. Each entry follows the format below. The file is human-readable and editable before running `/podcast apply <slug>`.

## File header

```
# Library Proposals — [Source Title]

Generated by /podcast skill on [YYYY-MM-DD] from a run against
[source-slug]. Target libraries: `quotes-library.txt`, `clinic-library.txt`,
`translations-glossary.md`.

To apply these proposals: run `/podcast apply [source-slug]` in Cowork.
The apply step:
  – Refuses if any target library has uncommitted git changes.
  – Shows a per-library unified diff.
  – Lets you accept all, accept some, or reject.
  – Commits applied changes with this source-slug in the message.
  – Re-runs journal-side quality gates after the commit.

To reject a specific proposal, delete its block from this file before
running apply. To edit a proposal, edit it in place — apply uses
whatever this file contains at run time.

Status: PROPOSALS — nothing has been applied yet.
```

## Body — three sections

```
============================================================
SECTION A: quotes-library.txt — [N] proposals
============================================================

[Each quote entry as below]

============================================================
SECTION B: clinic-library.txt — [N] proposals
============================================================

[Each clinic entry as below]

============================================================
SECTION C: translations-glossary.md — [N] proposals
============================================================

[Each glossary entry as below]
```

## Quote entry format

```
### Proposal Q-[NNN]

Quote: "[The exact quote, ≤50 words]"
Attribution: [Named figure]
Source: [Work title], Section [N], line [N]
Theme: [Memoir chapter theme this illuminates: Babu | Faith | Discipline | ...]
Dedup check: PASSED — no quote in quotes-library.txt scored ≥85% Levenshtein.
Quality criteria met:
  ✓ Explicit named attribution
  ✓ Under 50 words
  ✓ Non-aphoristic (illuminates a specific point, not a general truism)
  ✓ Not already covered by existing library entry on this theme

Proposed insertion location in quotes-library.txt: under theme heading "[theme name]"

Provenance footer to be appended:
  Source: [Work title], Section [N], podcast skill [YYYY-MM-DD]
```

## Clinic entry format

```
### Proposal C-[NNN]

Pattern: [One-sentence description of the cognitive/emotional/behavioral pattern]
Source: [Work title], Section [N], lines [N–M]
Memoir applicability: [Which chapter theme this connects to and why]
Psychological lens: [Cognitive | Emotional | Behavioral | Developmental | Relational]
Why this is psychologically useful: [One sentence — required tag]

Excerpt (≤100 words):
  "[The portion of the source text that demonstrates the pattern]"

Dedup check: PASSED — no entry in clinic-library.txt covers this pattern.

Provenance footer to be appended:
  Source: [Work title], Section [N], podcast skill [YYYY-MM-DD]
```

## Glossary entry format

```
### Proposal G-[NNN]

Term: [Original-script form, e.g., الإمام]
Phonetic: [English phonetic, e.g., al-Imaam]
Meaning: [Concise definition]
Language family: [Arabic | Hebrew | Sanskrit | ...]
First appearance: [Work title], Section [N], line [N]
Dedup check: PASSED — term not in translations-glossary.md.

Proposed insertion location: under language-family section "[Arabic-origin terms]"
```

## Borderline file (06b-borderline.md)

Candidates that did not meet the quality bar go to `06b-borderline.md` with a brief reason:

```
### Borderline Q-[NNN] — REASON: aphoristic / no clear attribution / theme overlap with existing entry

Quote: "[The candidate quote]"
Source: [Work title], Section [N]
Why borderline: [One-line explanation]
User decision: [PROMOTE to proposals / KEEP as borderline / DISCARD]
```

The user can promote a borderline candidate by moving its block from `06b-borderline.md` to `06-library-proposals.md` before running apply.

## What never appears in proposals

- Tier 2 read-only libraries (`incident-bank.md`, `biographical-context.md`, `voice-fingerprint.md`, `craft-techniques.md`, `thematic-arc.md`, `master-context.md`).
- Tier 3 untouched files (`locked-paragraphs.md`, `temporal-guardrail.md`, `memoir-rules-supplement.txt`, `chapter-status.md`, `journal-workflow-v2.md`, `quotes-workflow.md`).
- Chapter files in `chapters/`.

If the skill ever generates a proposal for any of those, that is a defect — quality gate must catch it.
