What this lens surfaces

Five distinct external trust boundaries, not three. NotebookLM is the quietest one — it ships data into Google with no audit trail and the only control is the operator's manual upload step. The dual-auditor design also means two LLM vendors (Anthropic + Google) see the bundle content, not one.

Hotspots

  • SEC-01Source PDFs leave the boundary three times per book (Azure OCR, Anthropic inference, Google NotebookLM). Each requires a separate DPA review.
  • SEC-02NotebookLM upload is manual and unlogged — no record of which bundle ID went to which Google account.
  • ARCH-01The Operator is a single point of failure for the publish step (Tier-2 gates require explicit approval, no second reviewer).
  • COST-01Anthropic + Google AI billed to two separate accounts — there is no unified cost ceiling across vendors.

What this lens surfaces

Five trust boundaries, two LLM vendors with credentials in Keychain, one (Anthropic) with opaque CLI-managed auth. All persistent state is filesystem JSON — no database, no managed transactions. plan-dashboard talks to Gemini independently of the pipeline, which is a second data egress path the audit team may not expect.

Hotspots

  • SEC-03claude -p CLI manages the Anthropic credential opaquely — the pipeline cannot enforce rotation, scope, or audit of that key.
  • SEC-04plan-dashboard contains a second egress path to Gemini (@google/generative-ai in package.json) — separate from the pipeline's gemini_api_key Keychain entry.
  • ARCH-02No database — state is JSON files (orchestrator-state.json, cost-ledger.jsonl, findings.jsonl) with fcntl locks. Single-host concurrency only; will not scale to multi-host without rewrite.
  • ARCH-03Watchdog is a separate bash process tracking the orchestrator by PID/mtime — no centralised observability, no metrics export.
  • COST-02Three concurrent windows (PHASE_0B_MAX_WORKERS=3) triple the Anthropic API surface area during refinement phases.

What this lens surfaces

Six phases are LLM-authoring (red/orange). Two are Azure OCR/translate (teal). Two human-review halts (09 series-plan, finalize). Cost concentration is 05 refine · 07 chapter-design · 08 enrichment · 11 per-chapter; everything else is cheap. The convergence loop on 11 is the largest unbounded cost driver and the only one with a per-chapter cap.

Hotspots

  • COST-03Per-chapter cost cap is $5 by default (per_chapter_cost_cap_usd) — there is no fleet-level or per-book ceiling. A pathological book × 30 chapters = silent $150.
  • COST-04Phase 07 chapter-design runs on Opus 4.7 over the whole book — single largest input-token line item per run.
  • COST-050g dual-auditor doubles inference cost per chapter (Claude + Gemini) by design — accepted trade-off but visible cost driver.
  • ARCH-04Two human-halt gates (09, finalize) are essential controls but they extend wall-clock time and require operator availability.
  • SEC-05The NotebookLM upload arc sits outside the pipeline — bundle integrity from finalize to Google is not verified by any automated gate.

L4 Credentials & Auth Boundary Map

Where every secret physically lives, which surface reads it, and what it authorises. This is usually omitted from architecture decks; it is usually the most valuable diagram in a real audit. Every keychain entry name below is real and present on the operator's Mac per docs/setup/azure-stack.md.

read / invoke (local) credential → external (managed) credential or upload outside pipeline control local store external trust zone

What this lens surfaces

Every Azure key sits in the Mac's login Keychain, scoped per-resource. The Gemini key is alongside them. The Anthropic credential is the exception — it sits inside the claude CLI's opaque store, outside Keychain, and the pipeline cannot enforce rotation or scope policy on it. The NotebookLM trust path has no automated auth at all.

Hotspots

  • SEC-06Per-machine Keychain model — every Mac has its own copy. No central rotation point until ENABLE_KEYVAULT=true; rotation is currently a per-host script run.
  • SEC-07Anthropic credential lives in the claude CLI's store — the pipeline cannot list it, scope it, or trigger rotation.
  • SEC-08NotebookLM uploads use the operator's personal Google account; there is no service account or audit trail.
  • SEC-09Two paths to Gemini share one key — pipeline auditor AND plan-dashboard. Compromise of one surface compromises both.
  • ARCH-05Azure resources retain the legacy journal-* naming despite repo rename. Cosmetic now, becomes a real blocker for any policy automation that filters by name.
  • OK-01Subscription keys never appear in tracked files; infra/azure/azure-config.env contains only resource names and the subscription ID. Verified — this is the most defensible piece of the current model.

Glossary — short reference

orchestrate_book.py
Deterministic Python driver that advances a single book through the 14 named phases. State in _system/orchestrator-state.json, atomic rename on write.
watch_orchestrator.sh
Bash watchdog spawned by the orchestrator on every resume. Tracks parent PID and subprocess mtime/size; kills runaway children.
podcast-orchestrator
Claude agent that handles LLM-authoring inside each phase. Invoked via claude -p from the _authoring/ package.
podcast-challenger
Semantic-quality reviewer. Runs the convergence loop (≤5 iterations) before any per-chapter bundle ships.
podcast-publisher
Gates G1–G7 verifier. Moves a slug from content/drafts/ to content/published/.
podcast-auditor
Post-merge regression sweep. Triggered after every merge into develop.
Phase 04 ocr-translate
Azure Document Intelligence (prebuilt-read) + Translator (v3.0). Only Azure phase that handles raw PDF bytes.
Phase 0g dual-auditor
Per-chapter bundle audit run by both Claude (audit_bundle.py) and Gemini (audit_bundle_gemini.py) in parallel.
convergence loop
3 outer × 5 inner iterations max per chapter. Capped at per_chapter_cost_cap_usd (default $5).
cost-ledger.jsonl
Append-only per-call cost log with fcntl LOCK_EX. Pricing table in _cost_ledger.py covers Claude 4.x and 3.x families plus Azure Doc Intel / Translator / Speech.
per-content branch
Typed branch off develop per piece of content (book/, doc/, lecture/, article/, letter/, interview/, fallback draft/). Names from _branching.py.
finalize halt
Pipeline halts before publish so the operator can review in podcast-reader and run any post-pipeline analysis. Resume authorises the publish step.
publish_to_library.py
Promotion script — copies drafts/<slug> to published/books/<slug>. Always Tier-2 (asks before running).
plan-dashboard
Astro + React + SQLite local SPA that visualises plan/architecture/debt YAMLs. Uses @google/generative-ai for planner inference — second egress path to Gemini.
podcast-reader
Astro app that reads content/drafts/<slug> for operator review during the finalize halt. Local-only.
ENABLE_KEYVAULT
Feature flag in infra/azure/azure-config.env. Currently false. Flip activates Azure Key Vault as the central source of Mac keychain entries — addresses SEC-06.
