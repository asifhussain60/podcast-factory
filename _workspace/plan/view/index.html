10. P5 bug post-mortem (was P0 in v2)

Symptom

Last session: orchestrator ran 19 minutes on Phase 0b, wrote 5 .in.md window files, produced 0 .out.md artifacts. claude -p child was not visible in pgrep.

Root cause

Headless claude -p invocations require explicit permission flags to perform file-write operations. Default behaviour is to refuse the write tool, returning rc=0 with a refusal in stdout. Orchestrator only inspects out_path.exists(), so the refusal is invisible — log shows NO ARTIFACT and proceeds to the next window.

Fix (P5.1 + P5.2)

--- a/scripts/podcast/_chunking.py
+++ b/scripts/podcast/_chunking.py
@@ -190,7 +190,12 @@ def _run_one_window(...):
     _atomic_write(in_path, body)
     proc = subprocess.run(
-        [CLAUDE_CMD, "-p", prompt],
+        [CLAUDE_CMD, "-p", prompt,
+         "--permission-mode", "acceptEdits"],
         timeout=600, capture_output=True, text=True,
     )
-    if not out_path.exists():
+    if proc.returncode != 0 or not out_path.exists() or out_path.stat().st_size == 0:
+        raise WindowFailed(
+            f"win-{idx:03d}: rc={proc.returncode} "
+            f"out_exists={out_path.exists()} stdout={proc.stdout[:200]}"
+        )

v3 note: The v2 plan treated this as a band-aid that the SDK migration would obsolete. v3 cancels the SDK migration — claude -p with --permission-mode acceptEdits is now the permanent mechanism, not a band-aid. P-1 (Claude Code primary) makes this the long-term design, not the stopgap.

11. Claude Code primary execution v3

Reverses the v2 plan's wholesale Anthropic-Python-SDK migration. Anchored by principle P-1 in the YAML.

Why Claude Code stays primary (P-1 rationale)

  • Agent runtime preserved. Claude Code provides tool use, file-edit, multi-turn refinement, and permission semantics that the skill design depends on. Re-implementing these on top of a raw Python SDK client is significant work for negligible gain.
  • Permission semantics are explicit. --permission-mode acceptEdits makes write-tool authorization a first-class CLI surface (and a one-line P5 fix).
  • Skill + agent files load natively. Skills and agents are designed for Claude Code; claude -p picks them up without translation.
  • Cost tracking is straightforward. Claude Code emits a usage summary at end of run; P6 parses it. No SDK migration required to get token counts into the ledger.

The Python SDK still has narrow side-channel uses (P6 helper, P8 status helper). It is NOT the workhorse for refinement, phonetics, chapter design, enrichment, framing, or building.

13. Dos & Don'ts (curated)

✅ DO

  • Always pass --permission-mode acceptEdits to claude -p with file-write tools P5
  • Verify artifact exists AND is non-empty after every LLM call P5
  • Use claude -p (Claude Code) as the workhorse for every refinement / phonetics / chapter-design / enrichment / framing / building call P-1
  • Write heartbeat every ≤30s from a daemon thread, atomic JSON P7
  • Parse claude -p stdout usage line into cost-ledger.jsonl P6
  • Flag any unsourced numeric enumeration as P0 BLOCKED P4
  • Apply the period-vs-modern reframe to any anachronistic gloss P4
  • Spawn long jobs as subprocesses of the API, never as in-process BackgroundTasks P12
  • Keep state.json + filesystem authoritative; SQLite is index/cache only P13

❌ DON'T

  • Don't replace claude -p with the Anthropic SDK as the workhorse — that's the cancelled v2 P3 path P-1
  • Don't trust subprocess rc=0 alone — always verify artifacts
  • Don't invent enumerations to fill in a "twelve X" claim the source doesn't enumerate P4
  • Don't gloss "seven continents" as the modern list when the period referent is the seven climes P4
  • Don't run orchestrator in FastAPI BackgroundTasks — kills resilience
  • Don't split PDFs until DI ceiling is actually hit P17
  • Don't conflate _workspace/ with content/
  • Don't mix journal and podcast — separate skills, agents, state, services
  • Don't add SQLite before P13 — JSON files suffice for current scale
  • Don't rename phases mid-flight — W5 prereq_gate enforces zero-in-flight before P15 runs
