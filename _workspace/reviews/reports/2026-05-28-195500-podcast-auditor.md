## P0 Findings (Stop-and-fix before any new work)

### AU-P0-1 — Pipeline driver cannot be imported (VERIFIED)

**File**: `scripts/podcast/orchestrate_book.py` line 101
**Error**: `NameError: name 'REPO_ROOT' is not defined`

The H2 partial REPO_ROOT unification session (commit `ca37aaf`, 2026-05-29) removed the
local `REPO_ROOT = Path(__file__).resolve().parents[2]` definition from
`orchestrate_book.py` as part of the "27 scripts unified" pass. However, the replacement
import on line 113 only imports three functions:

```
from _paths import content_dir as _paths_content_dir,
                  find_content as _paths_find_content,
                  relative_to_repo as _paths_rel  # comment says "REPO_ROOT"
```

`REPO_ROOT` itself is listed only in the comment, not in the import. The module then uses
`REPO_ROOT` 25 times (lines 101–2186) without ever defining or importing it.

**Impact**: Confirmed NameError on `python3 -c "import orchestrate_book"`. Two test
modules that import `orchestrate_book` fail at collection:
- `tests/e2e/test_full_pipeline.py` — collection error
- `tests/test_orchestrate_paths.py` — collection error

The entire pipeline is non-runnable. A real book orchestration attempt would crash at
module load time.

**Fix sketch**: Add `REPO_ROOT` to the `_paths` import on line 113:
```python
from _paths import REPO_ROOT, content_dir as _paths_content_dir, ...
```
One-line change. Verify with `python3 -c "import orchestrate_book"` and re-run
the full test suite.

### AU-P1-4 — H2 REPO_ROOT unification is incomplete (VERIFIED)

The H2 session log claims "27 pipeline scripts now import from _paths.py." Two gaps:

**Gap A — tighten_source.py still defines its own REPO_ROOT**
`scripts/podcast/tighten_source.py` line 114: `REPO_ROOT = SCRIPT_DIR.parents[1]`
This was not migrated. It uses `parents[1]` (two levels up from the script), which
is correct from that path but is not the canonical `_paths.REPO_ROOT`. Low risk
since the value is the same, but the DRY principle is violated.

**Gap B — 20+ phases/ scripts define their own REPO_ROOT**
All scripts under `scripts/podcast/phases/` (e.g., `p1_1.py`, `p2_2.py`,
`dor_halts.py`, `augmentation.py`) define `REPO_ROOT = Path(__file__).resolve().parents[3]`
locally. They need `parents[3]` because they are one level deeper than top-level scripts.
These were intentionally left out of the H2 pass (they need a different parent depth),
but they represent 20+ local REPO_ROOT definitions that diverge from `_paths.REPO_ROOT`.
If the directory structure ever changes, phases/ will silently drift.

**Impact**: The largest gap (orchestrate_book.py) caused the P0 above. The phases/ gap
is lower risk since all values are arithmetically equivalent today, but violates the
single-source-of-truth goal of H2.
