## VALIDATION RULES

### Rule 1: NO FORWARD REFERENCES
A person, event, or concept MUST NOT be mentioned before their narrative introduction.
- Ishrat: Cannot appear before her introduction scene. In Ch01, first mention is now at line 99 (proper intro). Forward-reference at old lines 71/85 FIXED April 11.
- Javeed Bhai: Cannot appear before his introduction (post-9/11 / Love ch)
- Nadeem Moosa: Cannot appear before his introduction (Karachi college / Man ch)
- The cousin: Cannot appear before her introduction (childhood / Love ch)
- Ex-wife: Cannot appear before marriage section (Marriage ch)
- Saima: Cannot appear before Dianetics section (Marriage ch)

### Rule 2: CROSS-CHAPTER REFERENCES MUST BE BACKWARD-LOOKING
When referencing a person or event from another chapter, the reference must be to something ALREADY told in a previous chapter (based on the chapter order: Intro → Man → Love → Marriage → future chapters).
- Ch01 (Man) can reference: Intro context only
- Ch02 (Love) can reference: Intro + Ch01 events
- Ch03 (Marriage) can reference: Intro + Ch01 + Ch02 events
- Future chapters can reference: all prior chapters

### Rule 3: INCIDENT OWNERSHIP

### Rule 4: DAD'S ADVICE REFERENCES
Dad's advice can ONLY reference events and people already introduced in the SAME chapter's narrative section. Dad cannot reference a person or event the reader has not yet encountered in that chapter.

### Rule 5: VALIDATION SCAN PROCEDURE
On every writing pass:
1. Read this timeline
2. For each person/event mentioned in a chapter, verify it has been INTRODUCED before that point
3. For cross-chapter references, verify the referenced event exists in a PRIOR chapter
4. Flag any violation as: "TEMPORAL VIOLATION: [person/event] referenced at [chapter:line] before introduction at [chapter:line]"
