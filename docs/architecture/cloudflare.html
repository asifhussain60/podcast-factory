How an iPad on the couch reaches Asif's Mac

The journal app runs on Asif's Mac. The AI features call an Anthropic proxy that also lives on the Mac, on localhost:3001. Cloudflare is the wire — DNS, TLS, an Access policy gate, and an outbound-only tunnel — that exposes those two surfaces at journal.kashkole.com and journal-api.kashkole.com without opening a single inbound port. No firewall holes. No origin certificates. Auth-gated by email PIN before any traffic touches the Mac.

Browser → Cloudflare edge → Access → Workers (or tunnel) → Express proxy → Anthropic API. Nine hops, all encrypted, all logged.
·· why a tunnel and not a port forward ··

Asif's Mac sits behind a residential ISP. There is no static IP, no incoming port allowed, no public DNS pointing at the machine. A Cloudflare tunnel inverts that — cloudflared makes outbound QUIC connections to Cloudflare's edge, and CF edge routes inbound user traffic through those existing connections. The Mac stays a fortress; Cloudflare is the postal forwarding service.

·· what the cloudflare layer doesn't do ··

It does not run the journal app — the Mac does. It does not call Anthropic — the Express proxy does. It does not OCR Arabic — that's the Azure stack. Cloudflare is the wire and the bouncer: TLS termination, DNS, Access policy enforcement, and the outbound tunnel. Everything substantive happens inside or beyond.
