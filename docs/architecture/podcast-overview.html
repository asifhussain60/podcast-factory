podcast · overview

We don't write podcasts. We steer NotebookLM.

The podcast skill takes a source in any format — book, PDF, audio recording, Word doc, slide deck, spreadsheet, transcript, plain text — normalizes it to text in one universal intake step, and turns it into two files that NotebookLM ingests to produce a strong two-host Audio Overview. We never write dialogue. We never record. We never generate audio. We load the gun.

·· the steering layer ··

The default two-host personas

NotebookLM's Audio Overview always uses two hosts. The framing file decides what they sound like.
·· host A ··

Curious Mind

Stand-in for the listener. Asks "why does this matter?", "wait, what does that mean in practice?", "but isn't there a tension with…?" Warm, plain language, occasionally surprised. Not naive — just unguarded.

·· host B ··

Scholar / Companion

Domain-knowledgeable. Provides context, traditional grounding, citations from the source. Calm, precise, willing to admit uncertainty. Never lectures.

·· the steering language ··

Phrases that reliably bend NotebookLM's output: "Slow down on…" (hosts spend more time on that beat), "Treat X as the central tension" (hosts return to X), "End on a question, not a conclusion" (produces open-ended landing). These go into the customize-prompt file, not the source.
