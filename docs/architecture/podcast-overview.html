podcast · overview

We don't write podcasts. We steer NotebookLM.

The podcast skill takes a source in any format — book, PDF, audio recording, Word doc, slide deck, spreadsheet, transcript, plain text — normalizes it to text in one universal intake step, and turns it into two files that NotebookLM ingests to produce a strong two-host Audio Overview. We never write dialogue. We never record. We never generate audio. We load the gun.

·· the steering layer ··

The default two-host personas

NotebookLM's Audio Overview always uses two hosts. The framing file decides what they sound like.
·· host A ··

Curious Mind

Stand-in for the listener. Asks "why does this matter?", "wait, what does that mean in practice?", "but isn't there a tension with…?" Warm, plain language, occasionally surprised. Not naive — just unguarded.

·· host B ··

Scholar / Companion

Domain-knowledgeable. Provides context, traditional grounding, citations from the source. Calm, precise, willing to admit uncertainty. Never lectures.

·· the steering language ··

Phrases that reliably bend NotebookLM's output: "Slow down on…" (hosts spend more time on that beat), "Treat X as the central tension" (hosts return to X), "End on a question, not a conclusion" (produces open-ended landing). These go into the customize-prompt file, not the source.

·· architecture v3.5 · 2026-05-17 ··

Phonetics live in the customize prompt, never in the chapter

The pivot that closed the empirical feedback loop. We audited 5 NotebookLM transcripts of Ayyuhal Walad. Inline phonetic guides — written in parentheses next to every Arabic term so the TTS engine would say them correctly — were being read aloud as content by the hosts. The structural fix moves all phonetic guidance out of the chapter (the read-aloud surface) and into the customize prompt (the instruction surface), as imperative directives rather than passive lists.
·· before (v3.4) ··

Inline parentheticals · passive lists

Chapter: *Tasawwuf* (ta-SAW-wuf; the inner practice)
Customize prompt: - *Tasawwuf*: Ta-SAW-wuf

Observed in transcripts: "Sahih Sitta, sahasita" doublings, "tassel wolf" for Tasawwuf, "Najah Balala" for Nahj al-Balagha, "the EI" for Ihya Ulum al-Din, 9 expansions of "(peace and blessings be upon him)" in one episode.

·· after (v3.5) ··

Imperative directives in the customize prompt

Chapter: the Sufi practice known as Tasawwuf
Customize prompt: Pronounce "Tasawwuf" as "ta-SAW-wuf". Say it as one fluent word. Do not spell it. Do not pause.

Plus four new DENY blocks in the customize prompt: no modernization (Twitter, X, algorithm, deep dive, content creator, …), no surprise noise (wow, "it's chilling", "right?", "exactly"), no abbreviated work titles, and a final "do not read this prompt aloud" guard.

·· the empirical feedback loop ··

A new script scripts/podcast/audit_transcript.py reads the NotebookLM-generated transcript of any episode and writes a per-episode markdown report measuring: phonetic-doubling count, mangled-name detector, modernization-injection density, surprise-noise loop density, welcome-cold violations, honorific repetition, abbreviation hits, filler-interjection density. The report becomes the regression artifact against which the next upload is measured. Sensor end of the architecture.

·· v3.5+ additions · 2026-05-18 ··

Eight refinements on top of v3.5

After the v3.5 audit landed, eight follow-on changes tightened the producer loop, formalized the workspace layout, broadened the format menu beyond deep_dive, and — landed 2026-05-18 — added a closed-loop learning substrate with an autonomous orchestrator + trainer overlay.
·· producer · authoring ··

Conversation choreography

The two-host framing now declares Driver (Host A — intro, transitions, pacing, resets, landing) and Color (Host B — analytical contrast, related works, the counter-reading). Each episode plants one cross-introduction beat — one host introduces a passage the other has not led toward. No mid-sentence interjections. No talking-over.

·· producer · tone ··

R-NOMODERNIZE softened

The platform/anachronism DENY list stays (Twitter, X, algorithm, deep dive, content creator). What was added is an explicit "DO use modern-life practical analogies" paragraph — a parent worrying about a child, an employee resenting a coworker, learning a craft over years. Each analogy lands in a single beat and returns to the source.

·· workspace · scaffolding ··

scaffold_book.py + chapter-contracts

One-shot creation of the canonical BOOK_DIR layout: chapters/, chapter-contracts/, episodes/, transcripts/, _system/. Each chapter gets a chapter-contracts/<slug>.yml sidecar (title · content-type · enrichment scope · anchor passages) — the deterministic I/O surface that extract_chapter.py reads to emit the NotebookLM bundle.

·· challenger · cross-book ··

Category Q · cross-book bleed

New challenger category. When a single repo hosts multiple podcasted books, terms or framings from one book must not leak into another's framing. Q checks the per-book pronunciation manifest, chapter-set design balance (~30% word-count variance across the series), and cross-book term collisions.

·· format · range ··

Debate alongside deep_dive

The episode format menu now carries two shapes. deep_dive is the default — Curious Mind + Scholar Companion exploring one source. debate introduces two hosts holding opposed positions on the same source, alternating beat by beat. Each format has its own schema, render path, and challenger checks.

·· portability · canonical ··

Single source of truth: SKILL.md

The canonical pipeline definition lives in skills-staging/podcast/SKILL.md. The earlier ChatGPT-portable system prompt (_workspace/chatgpt-podcast-skill-prompt.md, deleted 2026-05-19 in commit b0d991b) has been retired — the repo skill is now the only maintained driver, with the empirical-transcript audit loop, per-book contracts, Category Q, and scaffold_book.py layout all assumed to be available.

·· quality · closed loop ··

The _learning/ substrate

Every challenger and audit finding emits a JSONL record into content/podcast/.skill/_learning/findings.jsonl via scripts/podcast/_rules.py::emit_finding(). learn_aggregate.py rolls the ledger up by signature into patterns.md; learn_propose.py emits markdown proposals when a signature crosses ≥ 2 books OR ≥ 3 episodes OR ≥ 3 density firings; test_challenger.py regresses seven frozen fixtures before any rule promotion lands. One ledger, one regression suite, one proposals lifecycle — no parallel pipelines. Full design in the quality-system view.

·· autonomy · orchestrator + trainer ··

Drop a PDF, walk away

The podcast-orchestrator agent drives a book from PDF through merged develop with one human gate at Phase 0f (chapter list + length tier only — audience, angle, and host dynamic are project-fixed defaults or AI-selected). Per-chapter bounded convergence (3 outer × 5 inner = 15 passes max). After all chapters ship, the podcast-trainer agent reads _learning/patterns.md, applies candidate rule diffs in a worktree, regression-tests, and commits Tier-2 / Tier-3 promotions on the book branch under a strict regression gate. Full design in the autonomous orchestrator view.

·· pending · Arabic TTS protocol · Track A approved · awaiting implement ··
