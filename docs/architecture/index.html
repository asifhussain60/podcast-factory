How the writing rooms actually work

Two skills (one for the memoir, one for the podcast) share a single brain for Arabic pronunciation. Two infrastructure pillars (Cloudflare for the public app, Azure for OCR and translation) carry traffic and compute. Twelve views in total — pick a starting point. 2026-05-17: podcast architecture pivoted to v3.5 after a 5-transcript empirical audit of Ayyuhal Walad; phonetic guidance now lives in the customize prompt, never inline. 2026-05-18: conversation choreography (Driver vs Color roles), R-NOMODERNIZE softened with a "DO use modern-life analogies" paragraph, scaffold_book.py, per-chapter chapter-contracts/<slug>.yml, Category Q cross-book-bleed checks, a debate episode format. 2026-05-18 (later): a closed-loop learning substrate at content/podcast/.skill/_learning/ turns every challenger and audit finding into a single append-only JSONL ledger, a regression-gated proposals lifecycle, and a per-book health score; a podcast-orchestrator agent drives an entire PDF through the pipeline with exactly one human gate at Phase 0f, then invokes a podcast-trainer agent that promotes rule changes on the book branch only when seven frozen fixtures stay green. Arabic TTS protocol and the skill ROADMAP loaded by every producer and challenger run. Infrastructure pillars: Cloudflare tunnel for public access without inbound ports, and the Azure stack (rg-journal-ai) for Arabic OCR + English translation — both scripted, idempotent, and parameterized for future apps.
