·· what changed on 2026-05-17 ··

Before today, each skill carried its own copy of these rules. The podcast had a "Podcast Pronunciation Lexicon" inside the memoir's translations-glossary; the root of the repo carried a one-off pronunciation key file; each book had its own pronunciation guide. Today, the canonical layer lives at content/_shared/arabic/, both skills are wired to read it on every run, and the challenger agent has a new Loop C4 that audits substitution compliance. The legacy locations now defer to the shared brain.

·· pending · Conversational vs Classical mode · Track A approved · awaiting implement ··

The tracked protocol at content/podcast/.skill/handbook/arabic-tts-protocol.md (cold-start item 17 for both producer and challenger) proposes a binary Mode classification per term: Conversational (smooth Anglicized phonetic for casual integration — Inshallah → in-shah-LAH) versus Classical (hard-syllable Tajweed approximation for scripture and named historical figures — Quran → koor-AHN). The change adds a Mode column to each book's _system/pronunciation.md (optional on parse; absence defaults to Classical), promotes the TTS engineering rules (Q→K, Dh→TH, Kh→KH guttural, stress-cap, hyphen-glottal) to numbered imperatives in 01-tts-pronunciation-key.md, and renames the per-episode framing section ## Pronunciation → ## Phonetic Key (TTS Pronunciation). Challenger Loop C5 will gain a sub-check flagging scripture-citation terms misclassified as Conversational. The protocol is consulted on every chapter and every episode authoring run. R-PHONETICS-OUT remains intact — chapters stay free of inline parens.
